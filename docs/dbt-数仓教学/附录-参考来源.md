# 附录 · 参考来源(权威一手资料)

> 本课程的每一条 dbt 概念性结论,均来自 **dbt 官方文档(docs.getdbt.com)** 与 **dbt Labs 官方博客(getdbt.com/blog)** 的一手来源。
>
> 这些来源经过一轮 deep-research 的对抗性交叉验证:25 条核心断言全部以 **3-0 一致票**通过(3 个独立审阅者均无法证伪),0 条被驳回。下表按主题归类,方便你顺着官方文档深入。

## 分层架构与项目结构(How we structure our dbt projects)

| 主题 | 官方 URL |
|------|---------|
| 分层总览 | https://docs.getdbt.com/best-practices/how-we-structure/1-guide-overview |
| staging 层规范 | https://docs.getdbt.com/best-practices/how-we-structure/2-staging |
| intermediate 层规范 | https://docs.getdbt.com/best-practices/how-we-structure/3-intermediate |
| marts 层规范 | https://docs.getdbt.com/best-practices/how-we-structure/4-marts |

**已验证要点**(→ 对应 [第 05](05-步骤3-staging清洗层.md)/[06](06-步骤4-intermediate中间层.md)/[07 章](07-步骤5-marts业务聚合层.md)):
- staging **统一物化为 view**(下游总拿最新数据 + 不浪费存储),允许重命名/类型转换/基础计算(如分转美元)/分类归桶,**明确禁止 JOIN 和聚合**。
- intermediate 命名 `int_[entity]s_[verb]s.sql`(以动词为核心:pivoted/aggregated_to_user/joined 等),默认物化为 **ephemeral** 或独立 schema 的 view,**不暴露在生产 schema**。
- (本项目因 Trino 不支持 CREATE VIEW,以上 view/ephemeral 建议均改为 table —— 属平台约束下的偏离,见各章说明。)

## 物化策略(Materializations)

| 主题 | 官方 URL |
|------|---------|
| 物化机制总览 | https://docs.getdbt.com/docs/build/materializations |
| 可用物化类型 FAQ | https://docs.getdbt.com/faqs/Models/available-materializations |
| 物化最佳实践指南 | https://docs.getdbt.com/best-practices/materializations/1-guide-overview |
| 增量模型最佳实践 | https://docs.getdbt.com/best-practices/materializations/4-incremental-models |

**已验证要点**(→ 对应 [第 10 章](10-步骤8-物化策略与增量.md)):
- 内置**五种**物化:`view`(默认)/`table`/`incremental`/`ephemeral`/`materialized_view`;其中 view/table/incremental 是覆盖大多数场景的三种主要物化。
- 官方指导原则「**从最简单开始,按需逐级升级**」:view → (查询太慢) → table → (构建太慢) → incremental。

## 增量模型(Incremental)

| 主题 | 官方 URL |
|------|---------|
| 增量模型总览 | https://docs.getdbt.com/docs/build/incremental-models-overview |

**已验证要点**:增量模型仅处理自上次运行以来新增/变更的数据。支持的仓库用 `merge`(插入新行 + 更新已有行);不支持的用 `delete + insert`。

## 快照 / SCD Type 2(Snapshots)

| 主题 | 官方 URL |
|------|---------|
| 快照概念 | https://docs.getdbt.com/docs/build/snapshots |
| updated_at 配置 | https://docs.getdbt.com/reference/resource-configs/updated_at |
| dbt snapshot 命令 | https://docs.getdbt.com/reference/commands/snapshot |

**已验证要点**:snapshots 实现 SCD Type 2(用 `dbt_valid_from`/`dbt_valid_to` 记录历史)。两种策略:`timestamp`(推荐,基于 updated_at 列)和 `check`(比较 check_cols)。`unique_key` 必须真正唯一;须 `materialized: 'snapshot'`。

## 测试与数据质量(Data tests)

| 主题 | 官方 URL |
|------|---------|
| 数据测试 | https://docs.getdbt.com/docs/build/data-tests |

**已验证要点**(→ 对应 [第 08 章](08-步骤6-测试与数据质量.md)):
- 测试本质是**返回「失败行」的 SQL** —— 返回 0 行即通过。
- 两类:**singular**(`tests/` 下的 .sql)与 **generic**(YAML 里参数化、可复用)。
- 内置**四大通用测试**:`unique`、`not_null`、`accepted_values`、`relationships`。
